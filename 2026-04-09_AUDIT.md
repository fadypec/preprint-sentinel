## Critical Issues

### 1. Enrichment pipeline is fully sequential

**Location:** `pipeline/orchestrator.py:679-699`

Each paper makes 3 serial HTTP calls (OpenAlex, Semantic Scholar, ORCID). For 1000 papers at ~2s each, this takes 2+ hours instead of minutes.

**Fix:** Add `asyncio.gather()` with semaphore, matching the pattern already used in `_run_fulltext()`.

### 2. Dedup fuzzy matching loads candidates into memory

**Location:** `pipeline/ingest/dedup.py:139-186`

Streams all papers in a +/-14-day window and runs `rapidfuzz.fuzz.ratio()` on each. No database-level fuzzy matching.

**Fix:** Use PostgreSQL `pg_trgm` extension (already installed in migrations) for DB-level trigram similarity, or at minimum add a LIMIT to fail-fast on first match.

### 3. Session error handler lacks logging

**Location:** `pipeline/db.py:63`

Bare `except Exception: await session.rollback(); raise` with no logging. Impossible to diagnose transaction failures.

**Fix:** Add `log.error("session_error", exc_info=True, error_type=type(exc).__name__)` before rollback.

### 4. Accessibility: missing form label elements (WCAG 2.1 AA violation)

**Location:** `dashboard/components/paper-filters.tsx`, `dashboard/components/settings-form.tsx`

Filter dropdowns and settings inputs use `aria-label` only. WCAG 2.1 AA requires visible `<label>` elements or at minimum `<label>` associated via `htmlFor`.

### 5. Accessibility: heading hierarchy gaps

Multiple pages jump heading levels or use `<h2>` for card sub-headers that aren't true second-level headings. Screen readers can't build a proper document outline.

## Medium Severity Issues

### Efficiency and Performance

- **`orchestrator.py:618-634`**: All full-text results held in memory simultaneously. For 1000+ papers, memory spikes to 500MB+.
- **`dashboard/lib/queries/papers.ts:375-378`**: `totalIngested` count query runs separately on every full-text search. Should be cached.
- **`dashboard/components/analytics-charts.tsx`**: No `React.memo()` — chart components re-render unnecessarily.

### Security

- **No CSRF tokens** on API routes accepting PUT/PATCH (`dashboard/app/api/papers/[id]/notes/route.ts`).
- **Rate limiting is in-memory only** (`dashboard/lib/rate-limit.ts`) — resets on restart, won't work multi-instance.
- **Slack webhook placeholder visible** in password field via format-revealing placeholder text.

### Database

- **Missing index on `language` column** — used in WHERE clauses for non-English paper queries. Full table scans.
- **No connection timeout settings** in `pipeline/db.py`. Long-running queries hang indefinitely.
- **Pool size of 5** may be undersized for 4-6 concurrent pipeline stages.
- **Duplicate SQL execution** in `scripts/monitor_data_corruption.py:75` — same query runs twice.

### Code Quality

- **`dashboard/lib/utils.ts:40`**: Unsafe `as Dimensions` cast without runtime validation of object structure.
- **`dashboard/app/paper/[id]/page.tsx:31-40`**: Unsafe type assertion on `stage2Result`. If data differs, runtime crash.
- **No error boundaries** in dashboard — component throws crash entire page. Missing `error.tsx` files.
- **Retry logic duplicated** across 6 ingest/enrichment clients. Should be extracted to shared utility.

### Accessibility

- Color contrast on Tailwind utility colors (`slate-400` text) may not meet 4.5:1 AA ratio.
- Dialog/modal components need focus trap verification.
- No `aria-controls` on expandable audit trail buttons.
