## Security

### Verified Secure

- **SQL injection** — all queries use Prisma parameterised templates or SQLAlchemy bound params
- **XSS** — React auto-escapes; react-markdown renders only safe formatting; no `dangerouslySetInnerHTML`
- **Secrets** — `.env` properly gitignored, never committed to history (verified via `git log`)
- **Security headers** — X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy all set (`next.config.ts:6-13`)
- **Auth guards** — all 5 pipeline server actions have `requireAdmin()` (`actions.ts`)
- **CSRF** — implemented on `PUT /api/papers/[id]/notes`, `PATCH /api/papers/[id]`, `PUT /api/settings`
- **Input validation** — enum filters validated against whitelists (`papers.ts`), dates validated with regex (`actions.ts:29-35`)

### Issues Found

| # | Severity | Finding | Location |
|---|----------|---------|----------|
| 1 | HIGH | **CSRF missing on 6 routes** — `POST /api/pipeline`, `POST /api/pipeline/pause`, `POST /api/pipeline/resume`, `PUT /api/pipeline/schedule`, `POST /api/alerts/digest`, `POST /api/alerts/test` all lack `csrfCheck()` | `dashboard/app/api/pipeline/route.ts:38`, `pause/route.ts:4`, `resume/route.ts:4`, `schedule/route.ts:5`, `alerts/digest/route.ts:9`, `alerts/test/route.ts:4` |

## Efficiency

### Verified Efficient

- **HTTP clients** — all 9 ingest clients properly reuse httpx.AsyncClient via context managers
- **Enrichment** — parallelised with asyncio.gather + semaphore (4 sources concurrent)
- **Dedup** — batch DOI lookup via single IN query (`dedup.py:130-136`)
- **Dashboard queries** — paginated with LIMIT/OFFSET, count + data parallelised with Promise.all
- **Score sort** — COALESCE with JSONB dimension sum for papers with NULL aggregate_score

### Issues Found

| # | Severity | Finding | Location |
|---|----------|---------|----------|
| 1 | HIGH | **Unbounded memory on ingest** — loads ALL existing papers (DOIs, titles, versions) into memory with no pagination. At 500K+ papers this will exhaust RAM. | `orchestrator.py:559-565` |
| 2 | MEDIUM | **No timeout on asyncio.gather()** — 4 gather calls (fulltext, enrichment, translation x2) have per-task timeouts on HTTP but no aggregate timeout. A hung task blocks the entire stage. | `orchestrator.py:717, 840, 910`; `coarse_filter.py:166` |
| 3 | MEDIUM | **recharts in main bundle** — statically imported, adds ~400KB even on non-analytics pages | `analytics-charts.tsx:4-15` |
| 4 | LOW | **Index-based React keys on mutable arrays** — 3 components use array index as key for dynamic content (errors, sentences) | `run-history-table.tsx:200`, `methods-viewer.tsx:57`, `pipeline-progress.tsx:168` |

### Feature Suggestions Worth Considering

These would meaningfully improve the tool for biosecurity analysts:

1. **Paper provenance timeline** — show an analyst "this paper appeared on bioRxiv v1 → v2 → published in Nature" with timeline and change indicators. The DedupRelationship enum already has `PUBLISHED_VERSION` for this.

2. **Burst detection** — alert when multiple papers on the same dangerous technique appear within a short window from different groups. Could indicate coordinated research or a sudden field shift.

3. **Saved filter presets / threat scenarios** — let analysts save common filter combinations (e.g., "aerosol transmissibility + score >= 2") as named presets for one-click access.

4. **Per-dimension false positive tracking** — when an analyst marks a paper as FP, record which dimensions were over-scored. This gives data to eventually tune prompts without a dedicated analyst workflow.

5. **Comparative risk context** — show "this paper's score of 14/18 is in the top 2% of all assessed papers" so analysts can calibrate urgency.
