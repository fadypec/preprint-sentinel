# Codebase Audit — 2026-04-01

## Critical — Fix Before Hosting Online

### 1. No auth on API routes
Every API endpoint (`/api/papers`, `/api/pipeline`, `/api/settings`) has zero authentication checks. When auth is configured, the layout redirects unauthenticated users, but API routes remain wide open. Anyone could trigger pipeline runs (burning Anthropic credits), modify paper review statuses, or change pipeline settings.

**Files**: All `dashboard/app/api/*/route.ts`

### 2. Command injection risk in pipeline trigger
`dashboard/app/api/pipeline/route.ts:58-59` — `fromDate`/`toDate` from request body passed directly to `spawn()` args without validation. While `spawn` (not `exec`) mitigates shell injection, values should be validated as `YYYY-MM-DD`.

**Files**: `dashboard/app/api/pipeline/route.ts`

### 3. CSP blocks inline script in paper-filters
CSP header sets `script-src 'self'` which blocks inline scripts. The `<script dangerouslySetInnerHTML>` in `paper-filters.tsx` will silently fail in production. Need nonce-based CSP or external script file.

**Files**: `dashboard/next.config.ts`, `dashboard/components/paper-filters.tsx`

### 4. No Dockerfiles or deployment configuration
No Dockerfiles, no `vercel.json`, no Procfile. Need containerization for both Python pipeline and Next.js dashboard.

## High — Fix Before Running a Month of Data

### 5. LLM response validation missing
`pipeline/triage/coarse_filter.py:27-35` — malformed tool response defaults paper to passing (correct fail-open). But `methods_analysis.py:76-78` sets `risk_tier` to `None` if key missing, creating papers with null tier invisible in dashboard filters.

**Files**: `pipeline/triage/coarse_filter.py`, `pipeline/triage/methods_analysis.py`, `pipeline/triage/adjudication.py`

### 6. No composite database index for hot query path
Orchestrator queries `WHERE pipeline_stage = X AND is_duplicate_of IS NULL AND posted_date >= Y` repeatedly. Individual indexes exist but composite index on `(posted_date, pipeline_stage)` needed for month-long backfills with 100k+ rows.

**Files**: `pipeline/models.py`, Alembic migration needed

### 7. next-auth on beta version
`next-auth@5.0.0-beta.30` — pre-release software handling authentication. Pin to stable or accept risk.

---

## Medium — Fix for Public Deployment

### 8. No rate limiting on API routes
Single client can hammer API and database with unlimited requests.

### 9. Settings API stores Slack webhooks in plaintext
Settings form reads webhook from DB and sends to browser. Sensitive credentials should be write-only (display masked, only accept new values).

**Files**: `dashboard/app/api/settings/route.ts`, `dashboard/components/settings-form.tsx`

### 10. Dashboard .env.example missing
Pipeline has `.env.example` but dashboard has no documented env vars. Deployer must reverse-engineer what's needed.

### 11. No CI/CD
No GitHub Actions. Need at minimum: lint, type check, tests on push.

### 12. Dual ORM schema drift risk
SQLAlchemy and Prisma both define same schema independently. Alembic has migrations; Prisma has none. Adding a column via Alembic breaks dashboard until `schema.prisma` updated manually.

---

## Low — Polish

- No `output: "standalone"` in next.config.ts (needed for Docker)
- `findFirst` calls without `.take(1)` in a few places
- Overly broad `except Exception` in enrichment API clients
- Python deps use `>=` without upper bounds
- No monitoring/observability (Sentry, OpenTelemetry)
- PDF parsing silently swallows errors (`fulltext/retriever.py:229-243`)

---

## What's Good

- Pipeline architecture: 7 stages, error isolation per-paper, exponential backoff, structured logging
- No SQL injection: parameterized queries throughout (Prisma + SQLAlchemy)
- Secrets: Pydantic SecretStr, `.env` not in git
- Accessibility: semantic HTML, ARIA labels, `aria-live`, keyboard navigation
- Security headers: CSP, X-Frame-Options DENY, Permissions-Policy
- Cost design: Haiku bulk / Sonnet analysis / Opus adjudication only
- Skip-existing logic prevents re-processing on overlapping dates
- Prompt versioning + full audit trail on every LLM call
